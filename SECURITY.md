# Security Policy

## Supported versions

До первой стабильной версии security fixes выпускаются для последней опубликованной minor-линии.

| Version | Supported |
|---|---|
| 0.1.x | Yes |
| < 0.1 | No |

## Сообщение об уязвимости

Не создавайте публичный GitHub issue с exploit details.

Используйте GitHub Private Vulnerability Reporting в разделе **Security → Advisories → Report a vulnerability**. Если функция недоступна, свяжитесь с maintainer через приватный контакт, указанный в repository security settings (`SECURITY_CONTACT_PLACEHOLDER`).

Placeholder необходимо заменить перед публикацией. Здесь намеренно не указан несуществующий email.

Укажите:

- затронутую версию;
- минимальный reproduction;
- impact;
- возможный mitigation;
- допустимый способ обратной связи.

Maintainer подтвердит получение, согласует disclosure timeline и не будет требовать публикации exploit до готовности исправления.
