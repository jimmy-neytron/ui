# Архитектура

## Визуальное направление

Визуальный язык использует только общие принципы референсного календарного проекта: компактные контролы, почти чёрные нейтральные поверхности, тёплую светлую палитру, тонкие границы, сдержанные тени, спокойную типографику и небольшие или средние радиусы. Исходный код, структура, тексты, фирменные материалы и уникальные макеты не переиспользуются.

## Модель репозитория

Корень Git одновременно является корнем npm-пакета. Исходники библиотеки, playground, consumer-приложение, тесты, package-скрипты и workflows находятся в одном single-package repository. Через whitelist поля `files` в пакет проходят только `dist`, README, журнал изменений и лицензия.

## Публичная граница

- `src/index.ts` — корневой API.
- `src/components/*/index.ts` — direct imports.
- `package.json#exports` — контракт runtime, типов и стилей.
- `src/styles/index.css` — явная точка входа стилей.

Внутренние composables намеренно не экспортируются.

## Решения по компонентам

- Button использует нативный `<button>` и блокирует события через нативный `disabled` и явную защиту обработчика.
- Input и textarea используют нативные контролы и передают attrs непосредственно им.
- Стабильные id создаются через Vue `useId`, совместимый с SSR hydration.
- Select использует только примитивные значения, immutable controlled model и ARIA-паттерн combobox/listbox.
- Hidden inputs сохраняют стандартную семантику отправки HTML-форм.

## Размеры и focus

Отдельный density-режим отсутствует. Размеры `sm`, `md` и `lg` формируются через типографику, line-height и padding-токены, поэтому приложение может настроить компактность без фиксированных высот и дополнительных data attributes. Нативное browser glow подавляется только внутри компонентов; клавиатурный focus остаётся видимым как чёткая `:focus-visible`-обводка без размытой тени.

## Осознанные ограничения

Меню Select позиционируется локально и не использует Teleport или collision detection. Это сохраняет предсказуемые SSR-поведение, bundle size и управление фокусом, но контейнеры с clipping могут обрезать меню. В будущем внутренний positioning adapter сможет перемещать или измерять меню без изменения props и events.

Поиск локальный и синхронный. Владение асинхронными данными остаётся у приложения-потребителя.
